AWS DEA-C01: What Each Domain Actually Tests (Not What AWS Says)

Published March 4, 2026 · 5 min read

AWS describes the Data Engineer Associate (DEA-C01) exam with nice, clean domain descriptions. Here's what they DON'T tell you about what each domain actually tests.

Domain 1: Data Ingestion and Transformation (34%)

What AWS says: "Design and implement data ingestion solutions" and "develop data transformation solutions."

What it actually tests:

This domain is one-third of the exam and it's ALL about knowing when to use which service for data movement.

The big decision tree you need in your head:

  • Real-time streaming? → Kinesis Data Streams or MSK (Managed Kafka)
  • Near real-time ETL? → Kinesis Data Firehose
  • Batch ETL? → AWS Glue
  • Large-scale data migration? → DataSync, Snow Family, DMS
  • Database replication? → DMS with CDC (Change Data Capture)

The tricky questions pair two services that BOTH work but one is more appropriate. Example: "You need to ingest clickstream data with sub-second latency." Both Kinesis Data Streams and MSK work. The answer depends on: does the scenario mention existing Kafka expertise? If yes → MSK. If no → Kinesis (because it's fully managed and simpler).

Glue is EVERYWHERE. Know Glue ETL, Glue Crawlers, Glue Data Catalog, Glue Studio, Glue Streaming ETL. If you know Glue cold, you've got 15-20% of the exam locked down.

Domain 2: Data Store Management (26%)

What AWS says: "Choose appropriate data stores" and "manage data lifecycle."

What it actually tests:

This is the "which database?" domain. AWS has like 15 database services and you need to know the right one for every scenario.

Quick decision tree:

  • Relational + SQL? → RDS or Aurora
  • Key-value with microsecond latency? → DynamoDB
  • Document store? → DocumentDB
  • Graph data? → Neptune
  • Data warehouse / analytics? → Redshift
  • Time series? → Timestream
  • Caching? → ElastiCache (Redis or Memcached)

But the real test is understanding S3 storage classes and lifecycle policies. Questions like: "Data is accessed frequently for 30 days, occasionally for 90 days, then must be retained for 7 years." You need to design a lifecycle policy: S3 Standard → S3 IA → Glacier Deep Archive.

Also know: Lake Formation, data partitioning strategies in S3 (by date, by region), and Redshift Spectrum for querying S3 data directly.

Domain 3: Data Operations and Support (22%)

What AWS says: "Automate data processing" and "ensure data quality."

What it actually tests:

Pipelines and monitoring. Specifically:

  • Step Functions for orchestrating multi-step data pipelines
  • MWAA (Managed Apache Airflow) for complex DAG-based workflows
  • EventBridge for triggering pipelines on events
  • CloudWatch for monitoring with custom metrics and alarms

The exam loves asking: "A data pipeline fails at 3 AM. How do you detect and respond?" The answer always involves CloudWatch Alarms + SNS notifications + Lambda for auto-remediation.

Data quality questions focus on Glue Data Quality (new-ish feature), data validation rules, and how to handle schema evolution in your data lake.

Domain 4: Data Security and Governance (18%)

What AWS says: "Apply authentication, authorization, and encryption."

What it actually tests:

This is smaller but the questions are detailed:

  • Lake Formation permissions — this is the modern way to manage data lake access. Know fine-grained access control, tag-based access, and cross-account sharing.
  • KMS encryption — customer managed keys vs AWS managed keys, envelope encryption, key rotation
  • IAM policies for data services — S3 bucket policies vs IAM policies vs VPC endpoints
  • Data masking and PII detection — Macie for S3, Glue sensitive data detection

The governance questions are about data lineage, data catalogs, and ensuring compliance (GDPR, HIPAA) with encryption and access controls.
