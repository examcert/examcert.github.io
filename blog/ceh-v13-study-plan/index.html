The Exact 6-Week Study Plan That Got Me Through CEH v13

Published February 11, 2026 · 8 min read

I passed the CEH v13 in December 2025 with a score of 89%. Not because I'm some security genius — because I followed a structured plan and didn't deviate from it. Here's the exact schedule, resources, and strategies I used across six weeks.

Fair warning: this plan assumes you have basic IT knowledge (networking fundamentals, OS concepts). If you're completely new to IT, add 2-3 weeks for prerequisites.

Week 1: Reconnaissance and Scanning (Modules 1-5)

Study hours: 10-12 hours

The first five modules cover footprinting, scanning networks, enumeration, and vulnerability analysis.

What to learn:

  • Passive vs active reconnaissance techniques
  • DNS enumeration, WHOIS lookups, subdomain discovery
  • Nmap scan types (SYN, TCP connect, UDP, NULL, FIN, XMAS)
  • Network scanning methodology
  • Banner grabbing and OS fingerprinting

Hands-on this week:

  • Install Kali Linux in a VM
  • Run Nmap against a target on HackTheBox or TryHackMe
  • Practice WHOIS, nslookup, and dig commands

Practice questions: Do 50 questions using free CEH v13 practice exams on ExamCert.

Week 2: System Hacking and Malware (Modules 6-7)

Study hours: 10-12 hours

Password cracking, privilege escalation, maintaining access, and malware analysis.

What to learn:

  • Password attack types: dictionary, brute force, rainbow tables, pass-the-hash
  • Windows and Linux privilege escalation techniques
  • Rootkits, keyloggers, spyware
  • Fileless malware techniques
  • Anti-malware evasion

Key exam tip: Know these tools by name: Hashcat, John the Ripper, Mimikatz, Metasploit, Empire, Cobalt Strike.

Week 3: Network and Web Application Hacking (Modules 8-14)

Study hours: 12-15 hours (heaviest week)

This covers ~35-40% of the exam content.

What to learn:

  • ARP poisoning, MAC flooding, DHCP starvation
  • Social engineering techniques
  • OWASP Top 10 2021 — know ALL ten
  • SQL injection types: in-band, blind, out-of-band
  • XSS types: reflected, stored, DOM-based
  • CSRF, SSRF, XXE, command injection

Week 4: Cloud, IoT, and Cryptography (Modules 15-20)

Study hours: 10-12 hours

What to learn:

  • Cloud attack vectors: S3 bucket misconfigurations, IAM privilege escalation
  • IoT architecture and attack surfaces
  • Symmetric vs asymmetric encryption, hashing algorithms
  • PKI, digital signatures
  • Common crypto attacks

Week 5: Full Practice Exams

Study hours: 12-15 hours

No new material. Just exams.

  • Take at least 3 full-length practice exams (125 questions each)
  • Use ExamCert's CEH v13 question bank
  • Review EVERY wrong answer

My scores this week: 68% → 74% → 81%

Week 6: Weak Area Blitz + Exam

Days 1-3: Review wrong-answer notes. Focus on topics below 70%.

Days 4-5: One final practice exam. 80%+ = ready.

Day 6-7: Exam day.

Exam Day Tips

125 questions in 240 minutes (4 hours).

  • Don't second-guess — first instinct on tool-name questions is usually right
  • Watch for "BEST" and "FIRST" in questions
  • Flag and move on if a question takes >3 minutes
