GCP Professional Cloud Security Engineer: What Each Domain Actually Tests (From Someone Who Just Passed)

Published February 12, 2026 · 8 min read

I passed the GCP Professional Cloud Security Engineer two weeks ago. Scored in the "strong pass" range, which Google annoyingly doesn't give you an exact number for. But I know where I crushed it and where I barely survived, because the score report breaks down by domain.

Here's what each domain actually tests — not what the official exam guide says, but what showed up on my exam.

Domain 1: Configuring Access (about 25%)

What they actually test: The nuances of IAM that trip up experienced GCP users. Specifically:

  • IAM inheritance and deny policies. If you grant Editor at the org level and deny a specific permission at the project level, what happens? The deny wins, but only if the deny policy is correctly configured.
  • Service account impersonation chains. Service Account A impersonates Service Account B, which has access to BigQuery. What permissions does A need? Not BigQuery access — it needs roles/iam.serviceAccountTokenCreator on B.
  • Workload Identity Federation. Connecting external identity providers to GCP without service account keys. This came up 3-4 times on my exam.
  • Organization Policy constraints. Know the common constraint names.

My score: Strong. IAM is my day job. If you don't do GCP IAM daily, budget extra study time here.

Domain 2: Managing Network Security (about 20%)

  • VPC Service Controls dominate this domain. Creating service perimeters, configuring access levels, understanding ingress/egress rules.
  • Private Google Access vs Private Service Connect vs Private Service Endpoints. Three different things that sound similar. Know when to use each.
  • Hierarchical firewall policies vs VPC firewall rules. Priority ordering, inheritance.
  • Cloud Armor — WAF rules, DDoS protection.

My score: Moderate. VPC Service Controls tripped me up.

Domain 3: Ensuring Data Protection (about 20%)

  • CMEK vs CSEK. When to use each, how key rotation works, what happens if a key is destroyed. At least 4 questions on this.
  • Cloud DLP — identifying and redacting sensitive data, de-identification techniques.
  • Secret Manager — storing and rotating secrets, IAM for secret access.
  • Data residency — organization policies that restrict resource locations.

Domain 4: Managing Operations (about 20%)

  • Security Command Center (SCC) — Premium vs Standard tier, security findings.
  • Chronicle SIEM — log ingestion, detection rules (YARA-L). This caught me off guard with 3-4 questions.
  • Cloud Audit Logs — admin activity vs data access vs system event logs.
  • Binary Authorization — enforcing trusted container images in GKE.

Domain 5: Supporting Compliance (about 15%)

  • Assured Workloads — creating compliant environments for FedRAMP, HIPAA.
  • Access Transparency and Access Approval — understanding when Google personnel access your data.

What I'd Do Differently

  1. Spend 30% of study time on VPC Service Controls. I spent about 10%. That was a mistake.
  2. Learn Chronicle. Google is pushing hard to integrate Chronicle into the certification.
  3. Use practice exams earlier. ExamCert's GCP security engineer practice tests were the closest to the real exam format I found.

Study Resources That Actually Helped

  • Google Cloud Skills Boost security courses (free tier)
  • The official exam guide's linked documentation
  • ExamCert for domain-specific practice questions
  • Google's security best practices whitepapers
