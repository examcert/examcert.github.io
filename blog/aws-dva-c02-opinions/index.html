9 Opinions About the AWS Developer Associate That'll Probably Make You Mad

Published February 24, 2026 · 5 min read

I passed the AWS Developer Associate (DVA-C02) six weeks ago. Since then, I've had a lot of thoughts. Some of them are helpful. Some of them are petty. All of them are honest.

Here are 9 opinions about the DVA-C02 that I stand behind, even if Reddit disagrees.

1. The DVA-C02 is harder than the SAA-C03, and I'll die on this hill.

Everyone says the Solutions Architect Associate is the "gold standard" associate cert. And sure, the SAA-C03 is broad. But the DVA-C02 goes deep on things the SAA-C03 barely touches — Lambda optimization, DynamoDB access patterns, X-Ray tracing, CodePipeline/CodeBuild/CodeDeploy minutiae.

Breadth is uncomfortable. Depth is where things get hard. The DVA-C02 chooses depth.

2. You don't need to be a developer to pass it. But it helps if you've been yelled at by one.

The exam assumes you write code. Not complex code — you're not implementing algorithms. But you need to read Python and Node.js snippets, understand Lambda handler signatures, and know what an API Gateway mapping template looks like.

More importantly, you need to understand developer workflows. CI/CD pipelines, blue-green deployments, canary releases, rollback strategies. If you've ever been on a team that deploys to production, you've lived this content.

3. DynamoDB is at least 20% of the exam. That's not an exaggeration.

Partition keys, sort keys, GSIs, LSIs, capacity modes, DynamoDB Streams, optimistic locking with version numbers. The exam tests DynamoDB like it's a separate certification. I've seen estimates ranging from 15-25% of questions being DynamoDB-specific.

If you don't understand partition key design and how to model access patterns, you will fail. Not "might fail." Will fail.

4. Lambda is the other 20%. Together they're basically half the exam.

Lambda cold starts, execution context reuse, Lambda Layers, environment variables, /tmp storage, destinations, dead letter queues, provisioned concurrency. The exam expects you to architect serverless applications in your sleep.

DynamoDB + Lambda knowledge = 40% of the exam solved. That's not a study tip — that's a strategy.

5. CodePipeline/CodeBuild/CodeDeploy is the most boring section and you can't skip it.

I tried to skim the CI/CD section because I use GitHub Actions at work and figured "CI/CD is CI/CD." Wrong. AWS wants you to use their tools, and the exam tests their tools specifically.

Know the difference between in-place and blue-green deployments in CodeDeploy. Know what appspec.yml files do. Know how CodeBuild uses buildspec.yml. Know how CodePipeline orchestrates the whole thing.

Boring? Absolutely. Testable? Extremely.

6. X-Ray is weirdly important for an exam that's supposedly about development.

AWS X-Ray shows up in 5-8 questions. That's a lot for a tracing service. You need to know: segments, subsegments, annotations vs. metadata, sampling rules, and how X-Ray integrates with Lambda, API Gateway, and ECS.

I think AWS is pushing X-Ray adoption through the certification program. Cynical? Maybe. But study it regardless.

7. The SAA-C03 teaches you WHAT services exist. The DVA-C02 teaches you HOW they break.

This is the real difference between the two associate certs. Solutions Architect says "use SQS for decoupling." Developer Associate says "your SQS consumer is failing — explain the retry policy, DLQ configuration, and visibility timeout interaction that's causing message loss."

The DVA-C02 tests failure modes. How Lambda retry behavior differs for synchronous vs. asynchronous invocations. What happens when a DynamoDB write exceeds provisioned capacity. How API Gateway handles throttling.

If you can troubleshoot AWS services, you'll pass. If you can only architect with them, study more.

8. Practice exams matter more for the DVA-C02 than any other AWS cert I've taken.

Because the DVA-C02 tests specific API calls, configuration details, and code patterns, practice questions are the most efficient way to learn. You can read documentation all day, but until you've answered a question about the difference between Lambda Destinations and DLQs, the distinction won't stick.

9. Getting the DVA-C02 alongside the SAA-C03 sends a specific signal to employers.

Having just the SAA-C03 says "I can architect on AWS." Adding the DVA-C02 says "I can architect AND build on AWS." That combo is what actually gets you hired for cloud engineering roles, because companies need people who can do both.

If you only get one, get the SAA-C03 — it's more universally recognized. But if you want to stand out from the crowd of SAA-C03 holders (and there are a LOT of them), the DVA-C02 is your differentiator.

The Bottom Line

The DVA-C02 is 65 questions, 130 minutes, roughly $150. It's a harder exam than most people expect, but it's also one of the most practical AWS certs you can earn. Everything on the exam maps directly to real-world development work on AWS.

You've been warned.
